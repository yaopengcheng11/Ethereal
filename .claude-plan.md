# Plan: Filter Multi-Select Support

## Context

The Projects page (`Projects.tsx`) currently uses a single-select filter (`activeFilter: string = 'All'`). Users can only select one category/tag at a time. When admins share a filter link, only that single filter value is included. The user wants multi-select so shared links can show more content.

## Changes

### 1. `ethereal/src/pages/Projects.tsx`

**State change:** Replace single `activeFilter` string with a `Set<string>`:

```
- const [activeFilter, setActiveFilter] = useState('All');
+ const [activeFilters, setActiveFilters] = useState<Set<string>>(new Set());
```

Empty set = "All" (no filter applied). This is cleaner than carrying a special "All" value.

**Filter button click logic:**
- Click "All" → clear the set
- Click a tag/category → toggle it in the set (add if missing, remove if present)
- Visual: selected buttons get the white-bg active style; "All" is active only when set is empty

**Filtering logic (line ~207):**
```
if (activeFilters.size > 0) {
  result = result.filter(p =>
    Array.from(activeFilters).some(f => p.type === f || p.tasks?.includes(f))
  );
}
```
A project matches if it matches ANY selected filter (OR logic).

**Share handler (line ~114):**
Change the payload from `{ activeFilter, activeLetter, searchQuery }` to `{ activeFilters: Array.from(activeFilters), activeLetter, searchQuery }`. Update the `name` fallback to join selected filters.

**All references to `activeFilter` in this file must be updated:** button active state checks, the `useMemo` dependency arrays, letter navigation reset logic.

### 2. `ethereal-api-mock/server.js`

**POST `/api/admin/share-filter`** (line 589): No change needed — it stores `filter` as-is.

**GET `/api/share/:token`** (line 617): Update the filter application logic:
```
- if (filter.activeFilter && filter.activeFilter !== 'All') {
-   projects = projects.filter(p => p.type === filter.activeFilter || p.tasks?.includes(filter.activeFilter));
- }
+ const filters = filter.activeFilters || (filter.activeFilter && filter.activeFilter !== 'All' ? [filter.activeFilter] : []);
+ if (filters.length > 0) {
+   projects = projects.filter(p => filters.some(f => p.type === f || p.tasks?.includes(f)));
+ }
```
The fallback to `filter.activeFilter` provides backward compatibility with existing share links.

### 3. `ethereal/src/pages/SharePage.tsx`

No changes needed. SharePage receives pre-filtered `projects` from the backend and only applies `activeLetter` client-side. The multi-filter logic is handled server-side.
