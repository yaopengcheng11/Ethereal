Project Overview 项目概览

Ethereal Project is the official portfolio and content management system for E STUDIO, a visual effects studio specializing in cinematic imagery and generative visual content. Built with a dark, high-density, film-grade visual language, the platform provides a complete solution for showcasing VFX projects.

ModuleDescriptionPortRuntime
ethereal/ React SPA Frontend / React 前端应用 3000 Vite Dev Server
ethereal-api-mock/ Express Mock API Backend / Express 模拟后端 8080 Node.js

Cinematic Visual Design

Ultra-dark theme with custom typography (Inter / Plus Jakarta Sans / Space Grotesk) and micro dot-matrix background texture.

Project Showcase

Video/image gallery with cover preview, hover playback, fullscreen viewing, and file download capabilities.

Admin Panel

Full CRUD for projects, drag-and-drop multi-file upload, tag management (merge/delete), and user management.

RBAC Access Control

Three-tier role system (Super Admin / Admin / Guest) with invitation-based registration mechanism.

Bookmark System

Logged-in users can bookmark/unbookmark projects with a dedicated bookmarks page for quick access.

Shareable Filter Links

Bilingual Content

Responsive Design

Full mobile adaptation with hamburger menu and adaptive grid layouts for all screen sizes.

3.5 Configure Nginx / 配置 Nginx

Create config: sudo nano /etc/nginx/sites-available/ethereal

server {
    listen 80;
    server_name your-domain.com;  # Replace with your domain / 替换为你的域名

    # Frontend static files / 前端静态文件
    root /var/www/EtherealProject/ethereal/dist;
    index index.html;

    # Gzip compression / Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css application/json application/javascript
               text/xml application/xml text/javascript image/svg+xml;

    # Static asset caching / 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot|mp4|webm|mov)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
        try_files $uri =404;
    }

    # Project media assets (uploaded files) / 项目媒体资源
    location /asset/ {
        alias /var/www/EtherealProject/ethereal/public/asset/;
        expires 7d;
        add_header Cache-Control "public";
        client_max_body_size 0;
        proxy_buffering off;
    }

    # API reverse proxy / API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # File upload size limit (1GB) / 文件上传大小限制
        client_max_body_size 1024M;

        # Timeout for large file uploads / 大文件上传超时设置
        proxy_connect_timeout 600;
        proxy_send_timeout 600;
        proxy_read_timeout 600;
    }

    # SPA fallback / SPA 回退
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Security headers / 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
}

Enable the site and restart Nginx / 启用站点并重启 Nginx：

# Enable site / 启用站点
sudo ln -s /etc/nginx/sites-available/ethereal /etc/nginx/sites-enabled/

# Remove default site (optional) / 移除默认站点（可选）
sudo rm /etc/nginx/sites-enabled/default

# Test config / 测试配置
sudo nginx -t

# Reload Nginx / 重载 Nginx
sudo systemctl reload nginx

4. Deploy with Docker Compose / Docker Compose 部署

Recommended for containerized deployment, ensuring consistent environments. 容器化部署的推荐方式，确保环境一致性。

4.1 Backend Dockerfile

Create ethereal-api-mock/Dockerfile:

FROM node:18-alpine

WORKDIR /app

# Copy package files and install dependencies
COPY package.json package-lock.json* ./
RUN npm install --production

# Copy application code
COPY server.js ./

# Create data directory for JSON database
RUN mkdir -p /app/data

# Create asset directory (will be mounted as volume)
RUN mkdir -p /app/asset

EXPOSE 8080

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:8080/api/projects || exit 1

CMD ["node", "server.js"]

4.2 Frontend Dockerfile (Multi-stage)

Create ethereal/Dockerfile:

# Stage 1: Build / 构建阶段
FROM node:18-alpine AS builder
WORKDIR /app
COPY package.json package-lock.json* ./
RUN npm install
COPY . .
RUN npm run build

# Stage 2: Serve with Nginx / 用 Nginx 服务
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

8. Data Backup / 数据备份

The JSON database and uploaded assets should be backed up regularly. 生产环境中应定期备份数据库和媒体资源。

#!/bin/bash
# backup.sh — Run daily via cron / 通过 cron 每天执行
# crontab -e → 0 3 * * * /var/www/EtherealProject/backup.sh

BACKUP_DIR="/var/backups/ethereal/$(date +%Y%m%d)"
PROJECT_DIR="/var/www/EtherealProject"

mkdir -p "$BACKUP_DIR"

# Backup database / 备份数据库
cp "$PROJECT_DIR/ethereal-api-mock/data/db.json" "$BACKUP_DIR/db.json"

# Backup media assets (incremental) / 备份媒体资源（增量）
rsync -a "$PROJECT_DIR/ethereal/public/asset/" "$BACKUP_DIR/asset/"

# Keep only last 30 days / 只保留最近 30 天
find /var/backups/ethereal/ -maxdepth 1 -type d -mtime +30 -exec rm -rf {} \;

echo "[$(date)] Backup completed → $BACKUP_DIR"

10. Troubleshooting / 常见问题

API requests return 404 in production / 生产环境 API 返回 404

The Vite proxy only works in development. Ensure Nginx is configured to proxy /api/* to the backend. Vite 代理仅在开发模式下工作，需确保 Nginx 配置了反向代理。

File uploads fail / 文件上传失败

Check Nginx client_max_body_size is set to at least 1024M. 检查 Nginx 的 client_max_body_size。

# In the /api/ location block:
client_max_body_size 1024M;

Case-sensitive path issues on Linux / Linux 路径大小写问题

server.js line 18 references 'Ethereal' (capital E). If your directory is named ethereal (lowercase), update:

// server.js line 18 — change 'Ethereal' to match your directory name
const ASSET_DIR = path.resolve(__dirname, '..', 'ethereal', 'public', 'asset');

PM2 process crashed / PM2 进程崩溃

pm2 logs ethereal-api --lines 50   # View recent logs / 查看日志
pm2 restart ethereal-api            # Restart / 重启

Docker containers won't start / Docker 容器无法启动

docker compose logs backend    # Check backend logs
docker compose logs frontend   # Check frontend logs
docker compose down && docker compose up -d --build  # Rebuild
